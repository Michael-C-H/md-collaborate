# 在线多人协作 MD 编辑器（MVP）

## Why（意图）

- 背景：从零搭建一个 B/S 在线协作工具，团队需要一个共同编辑 Markdown 文档的空间。
- 动机：现有用户体系已就绪，可复用 SSO；只需补齐编辑与协作能力即可上线。
- 部署：局域网内私有部署，不暴露公网；并发用户 ≤ 100。

## What（范围）

### 做什么

- [ ] 实时多人协同编辑（光标/选区可见，冲突自动合并，Google Docs 风格）
- [ ] 文档树/文件夹组织（新建/重命名/移动/删除）
- [ ] 富 Markdown 渲染（表格、代码高亮、图片上传、Mermaid）
- [ ] 文档级权限（读/写/管理）
- [ ] 版本历史与恢复（快照式，可回滚到历史版本）
- [ ] 导入 / 导出（导入 .md 文件；导出 .md / PDF）
- [ ] SSO 集成（OIDC / OAuth2，对接现有用户体系）

### 不做什么

- [ ] 评论 / 批注（原因：纳入 V2）
- [ ] 全文搜索 / 收藏 / 跨文档发现（原因：纳入 V2）
- [ ] 自建用户/密码体系（原因：复用现有 SSO）
- [ ] 公网部署 / 多租户隔离（原因：局域网单实例使用）

## 初步技术方向

B/S 架构；前端富文本编辑器 + CRDT 实现实时协同；后端 WebSocket 同步 + 关系型库存元数据 + 对象/文件存储存历史快照；OIDC 接入现有身份源。规模目标按 100 并发设计，无需横向扩展。

## 成功标准

- [ ] 两人及以上同时编辑同一文档，秒级同步且无内容丢失
- [ ] 用户首次访问通过 OIDC 单点登录，无需另行注册
- [ ] 文档按文件夹组织，权限按文档独立控制
- [ ] 表格、代码、图片、Mermaid 可正常渲染与保存
- [ ] 可查看历史版本并一键恢复
- [ ] 可导入 .md，可导出 .md 与 PDF
- [ ] 局域网内 100 并发用户操作流畅
