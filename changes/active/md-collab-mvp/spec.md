# 在线多人协作 MD 编辑器 — MVP Spec

## Why

当前无内部协作文档工具。团队需要一个 B/S 在线 Markdown 编辑器，支持多人实时协同、文档组织、版本回溯，并对接现有用户体系（OIDC）。

**目标**：MVP 上线后，团队可以在局域网内通过 SSO 登录，多人实时共编 MD 文档，按权限分享，回滚历史，导入导出。

## What

### 功能需求

| 编号 | 功能 | 验收标准 | 优先级 |
|------|------|---------|--------|
| F1 | SSO 登录 / 登出 | 用户访问任意页面，未登录则跳 OIDC 授权页；授权后回调写会话，前端可获取当前用户的 user_id、姓名、邮箱、头像 URL；登出清除本地会话并跳 OIDC 登出端点 | P0 |
| F2 | 用户信息获取 | 系统不建本地用户表；展示用户名/头像/可分享用户列表时，统一调用 SSO 用户接口（封装为后端代理，含 5 分钟缓存） | P0 |
| F3 | 实时协同编辑 | 单文档同时编辑者 ≤ 10 人；超出后第 11 人进入只读；每位编辑者光标、选区、姓名标签独立可见；网络正常下他人编辑在 1 秒内可见；冲突无需人工干预 | P0 |
| F4 | 文档树管理 | 支持文件夹嵌套，最大 5 级深度；可新建、重命名、移动、（软）删除文件夹与文档；同级名称不可重复；名称长度 1~100 字符，禁止 `/ \ : * ? " < > \|` | P0 |
| F5 | 富 Markdown 渲染 | 编辑器实时渲染：标题、列表、引用、表格、链接、代码块（含语法高亮）、行内代码、图片、Mermaid 图、数学公式 | P0 |
| F6 | 图片上传 | 支持粘贴 / 拖拽 / 工具栏上传；格式 JPG/PNG/GIF/WebP/SVG；单图 ≤ 10 MB；上传成功后插入图片 MD 语法并显示 | P0 |
| F7 | 文档权限模型 | 每个文档独立权限，三级：读 / 写 / 管理；创建者默认管理者；只有管理者可分配权限；默认权限 = 创建者私有（其他人无权访问） | P0 |
| F8 | 分享面板 | 文档管理者可打开分享面板：搜索 SSO 用户、添加为协作者并指定权限、修改、移除 | P0 |
| F9 | 文件夹权限 | 文件夹仅控制本节点可见性（读权限）；文件夹下文档的权限独立设置，不自动继承 | P0 |
| F10 | 自动快照 | 编辑状态下，每 5 分钟生成一次快照（与上一版有差异才存）；自动快照保留 90 天后清理 | P0 |
| F11 | 手动快照 | 用户可随时手动生成快照并命名（1~50 字符）；手动快照永久保留，除非用户删除 | P0 |
| F12 | 版本浏览与恢复 | 文档可打开版本面板：按时间倒序列出所有快照（自动/手动标记、创建人、时间）；可查看某版本内容；一键恢复（恢复动作本身生成新快照） | P0 |
| F13 | 导入 .md | 支持单文件上传，亦支持上传 .zip 批量导入（按目录结构还原为文件夹/文档）；单文件 ≤ 5 MB，zip ≤ 50 MB | P0 |
| F14 | 导出 .md | 单文档导出为 .md 文件；图片以原 URL 引用（不打包） | P0 |
| F15 | 导出 PDF | 单文档导出为 PDF，含富渲染（表格 / 代码 / 图片 / Mermaid） | P0 |
| F16 | 软删除 / 回收站 | 删除文档/文件夹进入回收站，保留 30 天；用户可在回收站查看、恢复、彻底删除；过期自动物理清理 | P0 |

### 关键场景

#### 场景：未登录访问受保护页面
- **Given** 用户未登录，访问 `/docs/{id}`
- **When** 前端检测无有效会话
- **Then** 浏览器跳转到 OIDC 授权地址，参数 `redirect_uri` 携带原页面，授权完成后回到 `/docs/{id}` 且会话生效

#### 场景：两人同时编辑同一段文字
- **Given** 用户 A、B 均拥有文档写权限，已打开同一文档
- **When** A 在第 3 行末尾插入 "hello"，同时 B 在第 3 行开头插入 "world"
- **Then** 双方 1 秒内看到同一结果 "world{原内容}hello"，无内容丢失，无需手工合并

#### 场景：第 11 人进入正在被 10 人编辑的文档
- **Given** 某文档已有 10 个写权限用户在线编辑
- **When** 第 11 个写权限用户打开该文档
- **Then** 该用户以只读模式进入，顶部提示"当前同时编辑人数已达上限，您处于只读模式"，10 人中任意一人离开后此用户可获得编辑权（需主动点击"加入编辑"）

#### 场景：非协作者访问私有文档
- **Given** 用户 C 不在文档协作者列表中
- **When** C 通过 URL 直接访问文档
- **Then** 后端返回 403，前端显示"无访问权限"页

#### 场景：恢复历史版本
- **Given** 文档当前为 v100，用户具有写权限，版本面板中存在 v50
- **When** 用户点击 v50 → 恢复
- **Then** 文档内容回滚到 v50，同时生成新快照 v101（标记"由 v50 恢复"）；其他在线协作者编辑器内容同步更新到 v50

#### 场景：自动快照在无变更时跳过
- **Given** 文档处于编辑会话中，上一次快照后无任何内容变化
- **When** 5 分钟定时触发
- **Then** 不生成新快照，避免无效版本堆积

#### 场景：删除文档后恢复
- **Given** 用户删除了某文档，距删除 < 30 天
- **When** 用户在回收站点击"恢复"
- **Then** 文档恢复到原文件夹（若原文件夹已删，则恢复到根目录），状态回到正常

#### 场景：上传超大图片
- **Given** 用户拖拽一张 12 MB 的图片到编辑器
- **When** 客户端检测大小超过 10 MB
- **Then** 拒绝上传，提示"图片不能超过 10 MB"，不调用后端

### 不做的事

- 评论 / 批注、@提及（V2）
- 全文搜索 / 收藏 / 跨文档发现（V2）
- 移动端原生 App（仅响应式 Web）
- 离线编辑 + 后续同步（仅在线编辑）
- 自建用户 / 密码体系（依赖 SSO）
- 多租户隔离 / 公网部署 / 跨实例同步

## How

### 1. 正常流程

#### 1.1 首次登录
1. 用户访问任意 URL → 前端发现无会话 → 跳 OIDC 授权
2. OIDC 回调 → 后端拿到 user_id 等用户信息 → 写入服务端会话 → 重定向回原页面
3. 前端从 `/api/me` 获取当前用户信息渲染顶栏

#### 1.2 创建并编辑文档
1. 用户在文档树点击"新建文档"，输入标题，创建空文档
2. 默认创建者拥有"管理"权限，其他人无权访问
3. 用户进入编辑器，输入内容；前端通过 WebSocket 与后端同步 CRDT 操作
4. 每 5 分钟系统检查内容有无变化，若有则生成自动快照

#### 1.3 分享文档
1. 管理者打开分享面板，搜索 SSO 用户（后端代理 SSO 用户接口）
2. 选择用户、指定权限（读 / 写 / 管理），保存
3. 被分享者下次进入后即可按权限访问

#### 1.4 恢复版本
1. 用户打开"版本历史"面板 → 时间倒序看到全部快照
2. 点击某版本 → 右侧预览内容
3. 点击"恢复" → 当前文档替换为该版本内容 → 生成新快照（标记来源） → 在线协作者编辑器内容同步刷新

#### 1.5 导出 PDF
1. 用户菜单选"导出 PDF"
2. 后端将 MD 渲染为 HTML（含 Mermaid 转 SVG），再转 PDF
3. 浏览器下载 PDF 文件

### 2. 失败处理

| 失败场景 | 处理方式 |
|---------|---------|
| SSO 服务不可达 | 登录页提示"身份服务不可用，请稍后重试"；不让用户进入系统 |
| SSO 用户接口失败（查询协作者） | 分享面板用户搜索返回错误提示"暂时无法获取用户列表"；不影响已有协作 |
| WebSocket 连接断开 | 编辑器顶部红条提示"连接已断开，正在重连…"；本地编辑暂存到内存，重连后续传；3 次重连失败转只读 |
| 单文档并发达上限 | 第 11 人进入只读模式（见 F3） |
| 图片格式 / 大小不合法 | 前端拒绝并提示具体原因；不发请求 |
| 导入 .zip 包含非法名称 | 整个 zip 校验阶段失败，提示"第 X 个文件名称非法"，不部分导入 |
| 导出 PDF 渲染失败 | 返回 500，前端提示"导出失败，请稍后重试或反馈"；不留无效文件 |
| 恢复版本时文档正被他人编辑 | 弹窗提示"恢复将丢弃 N 位协作者的未保存编辑，是否继续？"；确认后强制覆盖 |
| 删除文件夹下含非空文档 | 一并软删除（连同子树进入回收站）；恢复时整树恢复 |
| 上传过程中网络中断 | 前端中断该次上传，提示"上传失败，可重试"；不留半成品 |

### 3. 边界与约束

| 边界 | 说明 |
|------|------|
| 网络 | 仅局域网部署，不暴露公网 |
| 总并发 | ≤ 100 在线用户 |
| 单文档并发编辑 | ≤ 10 人（超出转只读） |
| 文档树深度 | ≤ 5 级 |
| 文档名称 | 1~100 字符；禁止 `/ \ : * ? " < > \|` |
| 文档正文大小 | 单文档 ≤ 1 MB 纯文本（不含图片）|
| 单图片大小 | ≤ 10 MB；格式 JPG/PNG/GIF/WebP/SVG |
| 自动快照频率 | 每 5 分钟，无变化则跳过 |
| 自动快照保留 | 90 天后清理 |
| 手动快照 | 永久保留，仅用户主动删除 |
| 单文档手动快照上限 | 100 个（防止滥用） |
| 回收站 | 软删 30 天后自动物理清理 |
| 导入 zip | ≤ 50 MB；单文件 ≤ 5 MB |
| 浏览器支持 | 最新两个主版本 Chrome / Edge / Firefox |
| 用户信息来源 | 完全依赖 SSO 用户接口；本地不建 users 表，仅以 user_id 作为外键 |

## Impact

- **新建工程**：前端、后端、部署脚本，均为零代码起步
- **外部依赖**：现有 OIDC 身份服务（需提供：authorize、token、userinfo、用户搜索接口）
- **存储**：关系型库（元数据、权限、快照索引），文件存储（图片、可选快照正文）
- **运行时**：常驻 WebSocket 服务，需要支持长连接的反向代理配置

# Delta Spec — SSO 协议适配 & 超级管理员

> 基于: spec.md（含上一次 Delta：头像本地生成）
> 变更时间: 2026-05-13
> 变更原因: 取得 SSO 实际接口规格 — 非 OIDC，是自定义 ssoToken + verify 模式，且不提供用户搜索接口；同时启用 role=ADMIN 为系统超级管理员

## MODIFIED Requirements

### F1 SSO 登录 / 登出

**变更内容**：从 OIDC 标准协议改为自定义 ssoToken 验证模式。
**变更原因**：实际 SSO 不走 OIDC。

| 功能 | 验收标准（修改后） | 优先级 |
|------|------------------|--------|
| F1 | 三方系统跳转到本系统时在 URL 中带 `?ssoToken=xxx`；本系统后端调用三方 `/api/sso/verify` 验证 ssoToken，验证成功则建立本地会话（写入 `userId`、`username`、`displayName`、`role`），并 302 到原目标 URL（去除 ssoToken 参数）；登出 = 清除本地会话并跳转到三方系统首页（由三方提供地址，本系统无 end_session 调用）；ssoToken 为**一次性**，验证失败（如 `error_code: TOKEN_CONSUMED`、`TOKEN_EXPIRED`、`INVALID_TOKEN`）则显示"登录已失效，请从三方系统重新进入"页 | P0 |

#### 场景：携带 ssoToken 首次进入
- **Given** 用户在三方系统点击本系统入口，跳转到 `https://本系统/docs/abc?ssoToken=LEnBg0PQnAo00WmvxyKOLqmKVI4ppvia`
- **When** 本系统检测到 URL 含 ssoToken 且无本地会话
- **Then** 后端 POST 三方 `/api/sso/verify`，成功返回后建立本地会话，302 重定向到 `https://本系统/docs/abc`（去掉 ssoToken）

#### 场景：ssoToken 已被消费
- **Given** 用户刷新页面或重发同一带 ssoToken 的链接
- **When** 后端调用 verify 返回 `{status: 401, error_code: "TOKEN_CONSUMED"}`
- **Then** 若本地会话仍有效则忽略 ssoToken 进入；若无本地会话则显示"登录已失效，请从三方系统重新进入"页

### F2 用户信息获取

**变更内容**：用户信息**全部**来自 verify 接口的返回值；**移除**"调用 SSO 用户查询/搜索代理接口"的设计 — SSO 不提供该能力。本系统改为维护一张**已知用户缓存表**（`known_users`），记录所有曾通过 SSO 登录过的用户，供分享面板检索使用。该表不是用户体系，不存认证信息，仅作信息镜像。
**变更原因**：SSO 仅提供 verify 接口，无单独的 userinfo/搜索接口。

| 功能 | 验收标准（修改后） | 优先级 |
|------|------------------|--------|
| F2 | 用户每次通过 SSO 登录，verify 响应中的 `userId/username/displayName/role` 写入/更新本地 `known_users` 表（以 `userId` 为主键，`username` 唯一）；表用于：① 分享面板搜索；② 展示协作者姓名（避免每次都打 SSO）；③ 不用于鉴权 ← (was: 系统不建本地用户表) | P0 |

### F8 分享面板

**变更内容**：搜索方式从"调 SSO 搜索"改为"username 精确输入 + 本地 `known_users` 缓存检索"。
**变更原因**：SSO 无搜索接口。

| 功能 | 验收标准（修改后） | 优先级 |
|------|------------------|--------|
| F8 | 文档管理者打开分享面板，输入框支持两种模式：① 输入完整 username 精确匹配（如 `admin`）→ 在 `known_users` 中查找；② 输入姓名/username 关键字 → 在 `known_users` 中模糊匹配 `displayName` 或 `username`，返回前 20 条。若精确匹配的 username 不在 `known_users` 中，前端提示"该用户尚未访问本系统，请对方先登录一次后再添加"，**不允许**预先添加未登录过的用户 ← (was: 搜索 SSO 用户) | P0 |

#### 场景：精确添加已登录过的用户
- **Given** 用户 admin 此前曾登录本系统，已存在 `known_users`
- **When** 管理者在分享面板输入 "admin"
- **Then** 立即显示该用户卡片（displayName "系统管理员"），点击添加为协作者并指定权限

#### 场景：尝试添加未登录过的用户
- **Given** 用户 lisi 从未登录本系统，`known_users` 无此记录
- **When** 管理者输入 "lisi" 并尝试添加
- **Then** 前端提示"该用户尚未访问本系统，请对方先登录一次后再添加"，不创建协作者记录

## ADDED Requirements

### F19 超级管理员（基于 SSO role）

| 功能 | 验收标准 | 优先级 |
|------|---------|--------|
| F19 | verify 返回 `role: "ADMIN"` 的用户为本系统超级管理员，可：① 访问任意文档（视同管理者权限），② 强制删除任意文档/文件夹，③ 查看并恢复任意人的回收站项；非 ADMIN 用户的权限仍按文档协作者列表判定 | P0 |

## SSO 接口契约（确定版）

### POST `{SSO_BASE_URL}/api/sso/verify`

请求体（application/json）：

```json
{ "clientId": "...", "clientSecret": "...", "ssoToken": "..." }
```

成功响应（200）：

```json
{
  "userId": 1,
  "username": "admin",
  "displayName": "系统管理员",
  "role": "ADMIN",
  "resourceType": "ALGORITHM",
  "employeeType": "REGULAR"
}
```

失败响应（HTTP 200 含 status / 或 4xx，待 coding 阶段实测确认）：

```json
{ "status": 401, "error_code": "TOKEN_CONSUMED", "message": "SSO Token 已被使用或不存在" }
```

**字段使用约定**：

| 字段 | 本系统是否使用 | 用途 |
|------|--------------|------|
| userId | ✓ | 作为全系统用户主键（known_users.user_id、协作者外键、操作人外键、头像哈希源） |
| username | ✓ | 登录态展示、分享面板精确添加 |
| displayName | ✓ | 头像生成、协作者姓名展示、操作人展示 |
| role | ✓ | F19 超级管理员判定（仅 `ADMIN` 触发） |
| resourceType | ✗ | 本系统无相关场景，忽略 |
| employeeType | ✗ | 本系统无相关场景，忽略 |
